1. Fundațiile Designului Responsive

De ce este esențial designul adaptiv și cum funcționează

Ce este Designul Responsive?

Designul responsive înseamnă că site-ul tău se adaptează automat la dimensiunea ecranului dispozitivului. Un singur cod HTML/CSS care funcționează perfect pe mobil, tabletă și desktop.

De ce este important: 60% din trafic vine de pe mobil, Google penalizează site-urile non-responsive, UX mai bun pe toate dispozitivele.

Ingrediente principale: viewport meta tag, media queries, unități relative, layout-uri flexibile.

Principii Cheie

📱

Mobile-First Approach

Începe designul de la mobil și adaugă funcționalități pentru ecrane mai mari.

@media (min-width: 768px) pentru tabletă și desktop

📐

Unități Relative

Folosește % , em, rem, vw/vh în loc de px pentru dimensiuni adaptabile.

width: 100%, font-size: 1.2rem, padding: 2em

Viewport Meta Tag — Esențial!

Fără viewport
Cu viewport

Meta tag-ul viewport controlează cum browser-ul scalează pagina pe dispozitive mobile

<meta name="viewport" content="width=device-width, initial-scale=1.0">

2. Media Queries — Inima Responsive Design

Reguli CSS care se aplică doar când anumite condiții sunt îndeplinite

@media (max-width)

Se aplică când ecranul este mai mic decât valoarea specificată

1
2
3
@media (max-width: 768px) {
  .container {
    padding: 10px;
  }
}

@media (min-width)

Se aplică când ecranul este mai mare decât valoarea specificată

1
2
3
@media (min-width: 768px) {
  .grid {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}

Breakpoint-uri Populare

Dimensiuni standard pentru diferite dispozitive

📱 Mobile 320px - 767px
📱 Tablet 768px - 1023px
💻 Desktop 1024px+ 
/* Mobile First */
@media (min-width: 768px) { /* tablet */ }
@media (min-width: 1024px) { /* desktop */ }

@media (orientation)

Detectează orientarea dispozitivului

📱 Portret
📱 Peisaj
@media (orientation: portrait) {
  .sidebar {
    display: none;
  }
}

Condiții Multiple

Combină mai multe condiții cu and, or, not

min-width: 768px
AND
max-width: 1024px
@media (min-width: 768px) and 
       (max-width: 1024px) {
  .tablet-only {
    display: block;
  }
}

@media (prefers-color-scheme)

Detectează preferința utilizatorului pentru dark/light mode

☀️ Light
🌙 Dark
@media (prefers-color-scheme: dark) {
  body {
    background: #1a1a1a;
    color: #ffffff;
  }
}

4. Imagini Responsive

Imagini care se adaptează la toate dispozitivele și vitezele de conexiune

📱 Responsive Images cu srcset

Browser-ul alege imaginea potrivită pentru dispozitiv

<img src="small.jpg"
     srcset="small.jpg 480w,
             medium.jpg 768w,
             large.jpg 1024w"
     sizes="(max-width: 768px) 100vw,
            50vw"
     alt="Description">

🎨 Picture Element

Diferite imagini pentru diferite contexte

<picture>
  <source media="(max-width: 768px)"
          srcset="mobile.jpg">
  <source media="(min-width: 769px)"
          srcset="desktop.jpg">
  <img src="fallback.jpg" 
       alt="Description">
</picture>

📏 CSS max-width pentru Responsive

Imagini care nu depășesc containerul părinte

img {
  max-width: 100%; /* Nu depășește containerul */
  height: auto;    /* Menține proporțiile */
}

@media (max-width: 768px) {
  img {
    width: 100%; /* Full width pe mobil */
  }
}

🎭 Aspect Ratio cu CSS

Menține proporțiile fără să specifici height

.aspect-ratio {
  aspect-ratio: 16 / 9; /* 16:9 ratio */
}

.image {
  width: 100%;
  height: 100%;
  object-fit: cover; /* Crop pentru a umple */
}

💡 Pro Tip: Folosește loading="lazy" pentru imagini care nu sunt vizibile imediat:

<img src="image.jpg" 
     loading="lazy" 
     alt="Description">
